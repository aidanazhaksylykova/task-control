# TaskControl

Веб-приложение контроля исполнения задач по мотивам [prd.md](prd.md): единый реестр задач по проектам, самостоятельное обновление прогресса исполнителем и автоматические уведомления/эскалации при риске или факте срыва срока.

## Стек

- **server/** — Node.js + Express, встроенный `node:sqlite` (без нативной сборки), JWT-аутентификация, cron-движок уведомлений
- **client/** — React + TypeScript (Vite), React Router, recharts

## Запуск

```
cd server
npm install
npm run dev      # http://localhost:4000, при первом запуске сам создаст и заполнит БД демо-данными

cd client
npm install
npm run dev       # http://localhost:5173
```

Откройте http://localhost:5173 — на экране входа есть быстрый выбор демо-пользователей (пароль для всех: `password123`):

- **Айгуль Сатпаева** — руководитель (видит всё, управляет проектами/шаблонами/настройками)
- **Данияр Ахметов / Жанна Оспанова** — исполнители (видят только свои задачи)
- **Марат Жумабеков** — наблюдатель (только просмотр проекта «Ребрендинг сайта»)

## Что реализовано

- Дашборд, реестр задач с фильтрами, карточка задачи с зависимостями/историей/комментариями (§8.1–8.3 PRD)
- Аналитика: план/факт, % в срок, рейтинг надёжности, тепловая карта загрузки (§8.4)
- Единая лента уведомлений + движок автоматических триггеров: напоминания, просрочка, эскалация на N-й день, гибкая эскалация по бездействию — все интервалы настраиваются в «Шаблоны и настройки» → «Настройки проектов», а не зашиты в код (§7.2)
- Шаблоны повторяющихся задач, наблюдатели проекта (§8.6)
- Ролевой доступ: руководитель / исполнитель (только свои задачи) / наблюдатель (только просмотр)

Движок уведомлений проверяет триггеры каждые 2 минуты через `node-cron` (демо-частота) — либо нажмите «Запустить проверку уведомлений» на дашборде руководителя для мгновенного прогона.
